# PRISMA 2020 Compliance Checklist

Use this checklist at any phase to verify that all required sections are present and correctly labeled. A complete systematic review must address all 27 PRISMA 2020 checklist items.

## Critical Requirements for Full PRISMA 2020 Compliance

### Phase 1 Abstract
- [ ] Item 1: Title identification (includes "systematic review" and PICO)
- [ ] Item 2: Background/Rationale
- [ ] Item 3: Objectives (explicit PICO)
- [ ] Item 4: Eligibility Criteria
- [ ] Item 5: Information Sources
- [ ] Item 6: Risk of Bias Methods
- [ ] Item 7: Synthesis Methods
- [ ] Item 8: Study Selection and Counts
- [ ] Item 9: Synthesis Results (summary estimate, CI, I² or narrative consistency)
- [ ] Item 10: Limitations
- [ ] Item 11: Conclusions
- [ ] Item 12: Funding and Registration

All 12 abstract items must appear as **labeled subsections** — do not omit or merge items.

### Phase 2 Introduction
- [ ] Item 13: Rationale
- [ ] Item 14: Objectives (PICO format)

### Phase 3 Methods
- [ ] Item 3: Eligibility Criteria
- [ ] Item 4: Information Sources
- [ ] Item 5: Search Strategy
- [ ] Item 6: Study Selection Process
- [ ] Item 7: Data Extraction
- [ ] Item 8: Risk of Bias Assessment
- [ ] Item 10a: Outcome Data Items (primary and secondary outcomes, time points)
- [ ] Item 10b: Other Variable Data Items (participant/intervention/comparator/funding characteristics)
- [ ] Item 11: Effect Measures
- [ ] Item 13a: Studies Contributing to Each Synthesis
- [ ] Item 13b: Data Preparation and Transformation
- [ ] Item 13c: Methods for Tabulating Non-Pooled Results
- [ ] Item 13d: Statistical Synthesis Methods
- [ ] Item 13e: Methods for Exploring Heterogeneity
- [ ] Item 13f: Sensitivity Analyses
- [ ] Item 15: Certainty of Evidence Assessment (GRADE framework, SoF table planning)

All 6 Synthesis Methods sub-items (13a–13f) must be **explicitly labeled** — do not merge or omit.

### Phase 4 Results
- [ ] Item 16a: Study Selection (Flow Diagram)
- [ ] Item 16b: Excluded Studies (list with reasons)
- [ ] Item 17: Study Characteristics (table with author, year, design, demographics, intervention, outcomes)
- [ ] Item 18: Risk of Bias in Included Studies
- [ ] Item 19: Individual Study Results (per-study summary statistics and effect estimates)
- [ ] Item 20a: Characteristics of Studies in Each Synthesis
- [ ] Item 20b: Statistical Synthesis Results (summary effects, CI, heterogeneity measures)
- [ ] Item 20c: Results of Heterogeneity Investigations (subgroup/meta-regression results)
- [ ] Item 20d: Sensitivity Analysis Results (planned and post-hoc)
- [ ] Item 21: Reporting Bias Assessment (funnel plot, asymmetry test results)
- [ ] Item 22: Certainty of Evidence (GRADE ratings, Summary of Findings table with absolute effects per 1000 participants)

All 4 Synthesis Results sub-items (20a–20d) must be **explicitly labeled** — do not merge or omit.

### Phase 5 Discussion
- [ ] Item 23a: Interpretation of Findings (with reference to GRADE certainty ratings)
- [ ] Item 23b: Limitations of the Evidence Base (referencing GRADE domains)
- [ ] Item 23c: Limitations of the Review Process (search comprehensiveness, time-lag bias, etc.)
- [ ] Item 23d: Implications (for practice, policy, and research)

All 4 Discussion sub-items (23a–23d) must be **explicitly labeled** — do not merge or omit.

### Phase 6 Other Information
- [ ] Item 24a: Registration (register name, registration number, prospective/retrospective)
- [ ] Item 24b: Protocol Availability (where full a-priori protocol can be accessed)
- [ ] Item 24c: Amendments (list of changes with rationale and timing)
- [ ] Item 25: Financial Support & Conflicts of Interest (funding sources, author conflicts, funder veto authority)
- [ ] Item 26: Competing Interests (per-author declaration and management)
- [ ] Item 27: Availability of Data, Code, and Other Materials
   - [ ] Data extraction spreadsheet (DOI, repository URL)
   - [ ] Analysis code (GitHub/OSF link)
   - [ ] PRISMA flow diagram source file
   - [ ] SRDR+ reference
   - [ ] FAIR data principles compliance
   - [ ] Data license (CC-BY 4.0, CC0, etc.)

All 3 Registration sub-items (24a–24c) must be **explicitly labeled** — do not merge or omit.

## Handling "Not Applicable" Sections

If a section is genuinely not applicable to your review, write "**Not applicable**" as the section heading and provide a brief (1–2 sentence) justification. Do not omit any section.

Examples:
- Item 16b (Excluded Studies): "Not applicable — no studies were identified that appeared eligible based on title/abstract but were ultimately excluded at full-text assessment."
- Item 20c (Heterogeneity Investigations): "Not applicable — only one study met eligibility criteria; heterogeneity investigation was not possible."

## Special Requirements

### For Updated Reviews
Ensure these elements are explicitly addressed:
- Title: identify update status and cite original review
- Abstract Item 1: note update
- Introduction Item 14: cite original, specify updated aspects, explain rationale
- Phase 4 Item 17: distinguish newly included studies from those carried forward
- Discussion Item 23a: compare findings with previous version
- Phase 6 Item 24c: list any amendments

### For Living Reviews
Ensure these elements are explicitly addressed:
- Abstract Item 12: note living review status
- Discussion Item 23d: state planned update schedule and interim update triggers
- Phase 6 Item 24c: note that amendments will be documented at each update

## Verification Checklist

Before finalizing your review, verify:
- [ ] All 27 PRISMA 2020 items are addressed (see item list above)
- [ ] All required sub-items are explicitly labeled with their item number (e.g., "### 13a: Studies Contributing to Each Synthesis")
- [ ] No sections have been omitted; if truly not applicable, they state "Not applicable" with justification
- [ ] GRADE terminology is used throughout when discussing evidence quality
- [ ] Summary of Findings table is present with certainty levels and absolute effects per 1000 participants
- [ ] All cross-references between sections are accurate (e.g., results reference methods)
- [ ] For updated/living reviews: all update-specific requirements are met
